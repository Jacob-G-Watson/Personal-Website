React Web App

This was my first introduction into React or TypeScript. I was creating this project off of a specification from my university. It was a good opportunity to create something that I was able to show to friends and family as it had an easy to understand and comprehend frontend. The premise was to be able to create musical samples using different instruments with the library tone.js. In the web app, users will create music samples to be shared to a location (saved to an API).

  1. User A opens the web app.
  2. A landing page is displayed with a button to create a new sample.
  3. If the user has previously created samples, a list of samples is also displayed on the landing page, indicating whether it's been shared to a location (read via API).
  4. When creating a new sample, they give it a name, and select an instrument type for the sequence of notes.
  5. To build the sample, they toggle bars per note. This is played back to preview and saved within the app.
  6. Back on the landing page, the sample is shared to one of the preset locations (create via API) for use in the mobile app.

React Native App

Following straight on from my React Web App I moved into the native space. I worked in android and windows to create a mobile application. It made use of the device geolocation and camera, while integrating with public and private APIs. Coming from React Web the move to Native was relatively easy. In the mobile app, users can experience these music samples by physically visiting each location (read from the API).

  1. User B opens the mobile app.
  2. A map is displayed on the landing page, which they scroll around to find the nearest preset location (read via API).
  3. They physically move to that location (e.g. a location on University campus), the closeness being detected via geolocation.
  4. When they are within range of the location, they unmute the music for that location, which is a playlist derived from the samples made in the web app (read via API).
  5. They can customise their experience by adding their own “fingerprint” via device APIs.

Best Practice Premier (Bp Premier) is a practice management software for general practitioners and medical practices in Australia. It offers features such as clinical tools, decision making support, financial and management reporting, patient communication, and electronic prescriptions.

From a software and technical standpoint, Bp Premier is:

  • Built on Microsoft SQL Server for secure and efficient data access.
  • Runs on the latest Microsoft Client and Server Operating systems.
  • Scalable to suit practices of varying size and location.
  • Equipped with easy backup and restoration process.
  • Integrated with vetted third-party platforms through a secure API.
  • Updated monthly with free and easy to install data updates.
  • Compliant with industry standard security and functionality.

Game Frame Capture the Flag

I have made some recent contributions to the open source project, original developed by Steven Tucker, to help teach students core coding python concepts in a fun and competitive way. GF Capture The Flag is a competitive programming game. A competitor programs a set of autonomous bots to do battle in the Capture The Flag arena. The goal is to retrieve your flag from enemy territory before your enemy is able to retrieve theirs from you. Some bots may be attack bots, some may concentrate on defense, others may take on both roles. If caught in enemy territory (tagged by an enemy bot), you will be placed in jail. You can only escape from jail when a bot on the same team tags you.

Bots are programmed using the Python programming language. You can use any Python features to determine your strategy and next move, however the bots themselves must only be manipulated through the API provided in this document.
